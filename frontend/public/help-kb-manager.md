# Knowledge Base 使用說明

## 🚀 快速上手（5 步驟建立你的第一個知識庫 Bot）

不知道從哪裡開始？照這五步走，30 分鐘內完成從文件到 Bot：

| 步驟 | 做什麼 | 用哪個工具 |
|------|--------|-----------|
| **1. 整理文件** | 把 PDF 或手冊自動轉成 Q&A 格式 | **Doc Refiner** |
| **2. 下載 & 匯入** | 確認 Q&A 內容後，匯入知識庫（新建或現有皆可） | Doc Refiner → 下載 Q&A |
| **3. 確認就緒** | 回到本頁確認文件狀態為「就緒」 | **Knowledge Base**（本頁）|
| **4. 測試查詢** | 在右欄輸入問題，驗證 AI 是否能正確回答 | Knowledge Base 右欄 |
| **5. 部署 Bot** | 將知識庫設為「公司共用」，至 KB Bot Builder 建立 Bot 並取得連結 | **KB Bot Builder** |

> **選哪種知識庫？**
> - 需要 AI 整合多份文件、流暢回答 → 選「✨ AI 整合回答」
> - 需要 100% 原文呈現、答案不能被 AI 改寫 → 選「🎯 精確直答」

## Overview

**Knowledge Base** 是企業知識管理的核心工具，負責「建庫 → 維護 → 查詢」的完整流程。

| 對象 | 用途 |
|------|------|
| 一般員工 | 直接在右欄輸入問題，查詢公司知識庫（HR 政策、產品規格、操作手冊等） |
| 知識管理員 | 建立知識庫、上傳文件、管理文件版本與索引狀態 |
| 管理者 | 將知識庫設為公司共用，供 KB Bot Builder 引用部署為對外 Bot |

知識管理群組的三個 Agent 分工如下：
- **Doc Refiner** → 整理 PDF 為 Q&A 後匯入此處
- **Knowledge Base**（本 Agent）→ 管理文件、直接查詢
- **KB Bot Builder** → 以本 Agent 的知識庫為基礎，部署內部或對外 Bot

---

## 功能定位

Knowledge Base 是知識庫的建立與維護工具，讓使用者上傳文件、建立知識庫，並透過測試查詢驗證索引效果。建立好的知識庫可供 **KB Bot Builder** 引用，部署為對內或對外的 Bot。

---

## 1. 知識庫列表（左欄）

左欄分為兩個折疊區塊：

| 區塊 | 說明 |
|------|------|
| **我的知識庫** | 僅自己建立的個人知識庫，預設展開 |
| **公司共用** | 已設為公司範圍的知識庫，全公司可見，預設折疊 |

### 常見操作

- **新增知識庫**：點擊左上角「＋」，開啟設定視窗，填入名稱並選擇回答模式後點「建立」。
- **切換知識庫**：點選任一知識庫，中欄文件列表與右欄測試查詢即切換。
- **設定**：點「⋯ → 設定」，可調整名稱、AI 模型、回答模式、系統提示詞、知識庫範圍。
- **刪除**：點「⋯ → 刪除」，知識庫內所有文件同步刪除，**此操作無法復原**。

---

## 2. 回答模式

建立知識庫時，需選擇回答模式，決定 AI 如何回答問題：

| 模式 | 說明 | 適用情境 |
|------|------|---------|
| **✨ AI 整合回答**（RAG） | AI 從多份文件中整合資訊，生成流暢的完整回答 | 一般知識查詢、文件內容複雜多樣 |
| **🎯 精確直答** | AI 找到最符合的 Q&A 後，直接回傳原文答案，不改寫 | 標準 FAQ、需要 100% 忠實呈現答案的情境 |

### 精確直答模式說明

精確直答模式採用「LLM 選題 + 原文回傳」機制：

1. 系統用向量搜尋與關鍵字搜尋，找出最相關的候選 Q&A
2. LLM 從候選中判斷哪一題最符合使用者問題
3. 直接回傳資料庫中的原始答案，不經 AI 改寫

> 找不到相關 Q&A 時，系統會直接回答「找不到相關資訊」，而不是由 AI 猜測或生成。

> ⚠️ **精確直答模式限制**：只支援上傳 **FAQ 問答集**類型的文件。上傳時文件類型選項會自動鎖定為 FAQ。

## 3. 文件管理（中欄）

### 上傳文件

點擊「上傳文件」，支援以下格式：

- **PDF**（`.pdf`）
- **Word**（`.docx`）
- **純文字 / Markdown**（`.txt`、`.md`）

> 目前不支援 Excel、圖片等格式。

> 💡 **上傳前建議**：若文件屬於結構化知識點（員工手冊、SOP、法規、規格書），建議先使用 **Doc Refiner** 整理為 FAQ 格式再匯入，向量搜尋命中率更高，回答品質更好。一般參考資料或不需要問答化的文件則可直接上傳。

### 文件類型（切塊策略）

上傳時可選擇文件類型，影響切塊與檢索精準度：

| 類型 | 適用文件 | 備註 |
|------|---------|------|
| **FAQ** | 問答集、Q&A 對照表 | 最推薦；每對 Q&A 獨立切塊，語意最完整 |
| **規格（spec）** | 產品規格、技術文件 | 適合短段落、數字密集型 |
| **政策（policy）** | 法規、政策、合約 | 適合段落式說明文 |
| **文章（article）** | 一般文章、報告、提案書 | 預設類型 |
|| **對話（chat）** | 通訊軟體訊息、口語化短文 | Slack 同步內容建議選此類型；切塊較小，適合對話語境 |
| **參考（reference）** | 完整規章、不宜拆分文件 | 適合全文參考型 |

> 精確直答模式的知識庫只能上傳 FAQ 類型，其他類型按鈕會自動停用。

### 勾選文件

- **預設全選**：所有就緒文件均參與搜尋。
- 取消勾選可縮小搜尋範圍，提高精準度。
- 至少需勾選一份文件才能進行測試查詢。

### 文件狀態

| 狀態 | 說明 |
|------|------|
| 處理中 | 系統正在切塊與向量化（Embedding），請稍候 |
| 就緒 | 向量化完成，可參與檢索 |
| 錯誤 | 處理失敗（常見原因：PDF 加密、純圖片 PDF） |

## 4. 資料來源（自動同步連接器）

透過「資料來源」標籤，可將外部平台（目前支援 **Slack**）的內容自動同步至知識庫，無需手動上傳。

### 新增連接器

1. 切換至「資料來源」標籤，點擊「＋ 新增連接器」。
2. 輸入 **Slack User Token**（格式：`xoxp-…`），點擊「驗證並載入頻道」。
   - Token 需具備以下權限：`channels:history`、`channels:read`、`users:read`、`files:read`
   - 如何取得：前往 [api.slack.com/apps](https://api.slack.com/apps) 建立應用程式並安裝至工作區
3. 勾選要同步的頻道，設定連接器名稱、同步頻率與同步範圍後，點擊「建立並立即同步」。

### 連接器狀態

| 狀態 | 說明 |
|------|------|
| 同步中 | 正在從來源拉取資料並匯入知識庫 |
| 已啟用 | 運作正常，依排程自動同步 |
| 暫停 | 自動同步已停用，仍可手動觸發 |
| 錯誤 | 同步失敗，錯誤訊息顯示於卡片下方 |

### 連接器操作

| 按鈕 | 說明 |
|------|------|
| 立即同步 | 只拉取上次同步後的**新增**訊息（增量） |
| 重新同步 | 清除後重新拉取**所有**歷史訊息（完整，較耗時） |
| 編輯 | 修改頻道清單、連接器名稱、同步頻率等設定 |
| 暫停 / 啟用 | 切換自動同步排程 |
| 刪除 | 移除連接器（不會自動刪除已同步的文件） |

> **附件自動解析**：Slack 訊息中的 PDF 與 Word 附件也會被自動下載並解析匯入，無需額外操作。

> **文件類型建議**：Slack 同步的訊息內容建議選用「對話（chat）」類型，使用較小切塊策略，提昇對話語境的檢索精準度。

## 5. 段落內容編輯（Chunk Editor）

文件狀態為「就緒」後，點擊文件列表中的 ✏️ 圖示，即可開啟 **Chunk Editor**，查看並編輯該文件的所有切分段落（chunk）。

### 可以做什麼

| 操作 | 說明 |
|------|------|
| **編輯段落** | 點擊段落右上角鉛筆圖示，修改段落內容，儲存後自動重新 Embedding 與關鍵字索引（BM25） |
| **刪除段落** | 刪除不需要的段落；文件的段落數量自動更新 |
| **新增段落** | 點擊底部「＋ 新增段落」，輸入新內容後送出，系統自動 Embedding |
| **匯出 txt** | 點擊標題列的下載圖示，將所有段落匯出為 `.txt` 備份，可重複使用 |

### 權限

| 知識庫範圍 | 可編輯段落的使用者 |
|-----------|------------------|
| 個人私有（personal） | 知識庫建立者本人 |
| 公司共用（company） | manager 以上 |

## 6. 測試查詢（右欄）

- 選取知識庫並勾選文件後，在右欄輸入問題即可測試查詢效果。
- **AI 整合回答模式**：AI 回答附帶**來源標注**（來源：檔名），方便確認答案是否來自正確文件。
- **精確直答模式**：顯示「以下可能是你需要的答案：」並列出完整 Q&A 原文；找不到時直接提示找不到。
- 對話記錄會保留，離開頁面再回來仍可查看。
- 點擊右上角重新整理圖示可清除本段對話。

---

## 7. 知識庫設定

點「⋯ → 設定」或新增知識庫時，可調整以下項目：

| 設定項目 | 說明 |
|---------|------|
| **名稱** | 知識庫顯示名稱 |
| **LLM 模型** | 指定此知識庫使用的 AI 模型，留空則採系統預設模型 |
| **知識庫範圍** | 個人私有 / 公司共用（需 manager 以上才能變更） |
| **回答模式** | AI 整合回答（RAG）或精確直答，建立後可修改 |
| **自訂系統提示詞** | 覆蓋預設行為（僅 AI 整合回答模式有效），如指定回答角色、語氣、輸出格式 |

> 系統提示詞留空時，採用預設知識庫問答提示詞（含來源標注、Markdown 輸出）。

## 8. 知識庫範圍說明

| 範圍 | 誰能看到 | 典型用途 |
|------|---------|---------|
| 🔒 個人私有（personal） | 僅建立者本人 | 草稿、個人筆記、測試用 KB |
| 🏢 公司共用（company） | 同公司全體員工 | HR 規範、產品手冊、公司 FAQ |

| 操作 | member | manager | admin / super_admin |
|------|--------|---------|---------------------|

| 修改他人的知識庫 | ❌ | ❌ | ✅ |
| 刪除自己的知識庫 | ✅ | ✅ | ✅ |
| 刪除他人的知識庫 | ❌ | ❌ | ✅ |
| 上傳 / 刪除文件 | ✅（自己的 KB） | ✅ | ✅ |

| 測試查詢 | ✅ | ✅ | ✅ |

---

## 注意事項

- 刪除知識庫將一併刪除其所有文件，此操作**無法復原**。
- 上傳大型文件（> 10 MB）可能需要較長時間完成向量化，狀態顯示「處理中」時請耐心等待。
- **公司共用**的知識庫可被 KB Bot Builder 引用，建議由管理員統一維護品質與內容正確性。
- 精確直答模式適合內容固定、需要嚴格忠實呈現的 FAQ 場景；若知識庫內容較為多樣，建議使用 AI 整合回答模式。
