# Doc Refiner 使用說明

## Overview

**Doc Refiner** 是知識庫的「前置整理工具」，將現有 PDF 文件自動轉換為結構化 Q&A，讓文件知識快速進入知識庫並發揮效用。

| 典型使用情境 | 說明 |
|-------------|------|
| 員工手冊 / HR 規章 | 上傳 PDF → 整理成 Q&A → 匯入知識庫 → 員工自助查詢 |
| 產品規格書 | 快速萃取規格重點為問答對，供業務或客服查詢 |
| 培訓教材 | 將課程內容整理為問答集，用於測驗或知識庫 |
| 法規 / 政策文件 | 把複雜條文轉為易查詢的 Q&A 格式 |

知識管理群組的三個 Agent 分工如下：
- **Doc Refiner**（本 Agent）→ 整理 PDF 為 Q&A 後匯入知識庫
- **Knowledge Base** → 管理文件、直接查詢
- **KB Bot Builder** → 以知識庫為基礎部署 Bot

---

## 功能定位

Doc Refiner 以 AI 自動將 PDF 文件整理為結構化的 **Q&A 問答集**，整理結果可直接編輯後匯入知識庫、下載為 PDF，或下載為純文字檔。

---

## 使用流程

### 第一步：上傳文件

| 項目 | 說明 |
|------|------|
| **支援格式** | PDF（最大 20 MB） |
| **上傳方式** | 拖曳至上傳區，或點擊選擇檔案 |
| **使用模型** | 留空時使用租戶預設模型 |

點擊「**開始智慧整理**」，AI 將自動閱讀 PDF 並逐段輸出結果。

> 文件較大時會分段處理，右側標題列會顯示「第 N/M 段」進度，結果會逐步出現，不需等待全部完成。

---

### 第二步：比對 & 編輯

- **左側** 顯示原始 PDF 預覽，方便對照確認。
- **右側** 顯示 AI 整理結果，每一條均可直接點擊編輯。

| 操作 | 說明 |
|------|------|
| **編輯文字** | 直接點擊問題／答案欄位修改，離開欄位後自動儲存 |
| **刪除條目** | 滑鼠移至卡片右上角，點「🗑」刪除 |
| **新增條目** | 點底部「＋ 新增一條」 |
| **重新上傳** | 點左側「重新上傳」，回到第一步 |

---

### 第三步：匯出或匯入

整理完成後，右側頂部提供三個按鈕：

| 按鈕 | 用途 |
|------|------|
| **📥 匯入至知識庫** | 直接將 Q&A 條目推送至現有知識庫，或當場建立新知識庫；推薦做法 |
| **下載 TXT** | 下載 FAQ 格式純文字檔，可手動上傳至知識庫管理員，也可用文字編輯器修改 |
| **下載 PDF** | 下載排版精美的 PDF，供對外分享或列印 |

---

## 注意事項

- 掃描式 PDF（純圖片）無法提取文字，請使用含文字層的 PDF，或先使用 OCR Agent 處理。
- 文件包含表格時，表格會自動轉為結構化格式供 AI 理解，整理結果更準確。
- 整理結果僅供參考，建議人工審閱確認後再匯入知識庫。
- 長文件（逾 20,000 字）會自動切段分批整理，各段結果將依序累積顯示。
