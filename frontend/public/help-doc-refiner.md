# Doc Refiner 使用說明

## Overview

**Doc Refiner** 是知識庫的「前置整理工具」，將文件或筆記自動轉換為結構化 Q&A，讓知識快速進入知識庫並發揮效用。

| 典型使用情境 | 說明 |
|-------------|------|
| 員工手冊 / HR 規章 | 上傳 PDF / TXT → 整理成 Q&A → 匯入知識庫 → 員工自助查詢 |
| 產品規格書 | 快速萃取規格重點為問答對，供業務或客服查詢 |
| 培訓教材 | 將課程內容整理為問答集，用於測驗或知識庫 |
| 會議記錄 / 筆記 | 貼入非結構化文字，AI 主動提問並整合為 Q&A |
| 法規 / 政策文件 | 把複雜條文轉為易查詢的 Q&A 格式 |

知識管理群組的三個 Agent 分工如下：
- **Doc Refiner**（本 Agent）→ 整理文件 / 筆記為 Q&A 後匯入知識庫
- **Knowledge Base** → 管理文件、直接查詢
- **KB Bot Builder** → 以知識庫為基礎部署 Bot

---

## 兩種輸入模式

透過左側 sidebar 切換：

| 模式 | 說明 | 適用情境 |
|------|------|---------|
| **文件 → FAQ** | 上傳 PDF 或 TXT 檔案，AI 自動逐段抽取重要知識點並生成 Q&A | 員工手冊、規格書、法規、合約 |
| **筆記 → FAQ** | 直接在文字框貼入筆記或非結構化文字，AI 主動提問並整合碎片資訊 | 會議紀錄、口述重點、草稿、隨手筆記 |
| **SOP → FAQ** | 上傳 PDF 或 TXT 的 SOP 文件，AI 嚴格保留所有步驟細節生成 Q&A | 操作手冊、標準作業程序、流程說明 |

> **文件 → FAQ** 與 **SOP → FAQ** 的差異：前者著重萃取知識點，允許 AI 整合相近內容；後者嚴格保留每一個步驟，不合併、不省略，確保流程完整性。

---

## 使用流程

### 第一步：選擇模式與模型

- 在左側 sidebar 點選輸入模式（**文件 → FAQ**、**筆記 → FAQ** 或 **SOP → FAQ**）
- Sidebar 底部可切換 **AI 模型**，文件整理與後續改寫皆使用同一個選定模型

---

### 第二步：輸入內容

**文件模式 / SOP 模式：**

| 項目 | 說明 |
|------|------|
| **支援格式** | PDF、TXT（最大 20 MB） |
| **上傳方式** | 拖曳至上傳區，或點擊「上傳」按鈕選擇檔案 |

點擊「**開始整理**」後，AI 會自動閱讀文件並逐段輸出 Q&A。SOP 模式會額外確保每個步驟完整保留，不合併不省略。

> 文件較大時會分段處理，標題列會顯示「第 N/M 段」進度，結果逐步出現，無需等待全部完成。

**筆記模式：**

將文字貼入左側「原始筆記」區域，點擊標題列的「**轉成 Q&A**」，AI 會主動從非結構化內容中提取問題與答案。

> 掃描式 PDF（純圖片）無法提取文字，請先使用 OCR Agent 處理，或使用含文字層的 PDF。

---

### 第三步：審閱與編輯

- **文件模式左側** 顯示原始文件預覽，方便對照確認
- **右側** 顯示 AI 整理的 Q&A 卡片，每一條均可直接編輯

| 操作 | 說明 |
|------|------|
| **編輯文字** | 直接點擊問題／答案欄位修改 |
| **刪除條目** | 點卡片右上角刪除圖示（🗑）|
| **新增條目** | 點底部「＋ 新增一條」（有現有 Q&A 時顯示）|
| **清空全部** | 點 Q&A 標題列的「清空」按鈕（需確認）|

---

### 第四步：AI 改寫（選用）

AI 生成的 Q&A 不一定百分之百符合需求，改寫功能讓你針對單條 Q&A 微調，不需重新整理整份文件。

1. 點擊卡片右上角紫色改寫圖示（✦），展開改寫面板
2. 選擇**預設指令**（可多選，會自動串接）：
   - **問題更自然**：讓問題語氣更口語、貼近真實對話
   - **答案更清楚**：讓答案邏輯更清晰、易於閱讀
   - **答案更精簡**：刪除冗餘，保留核心資訊
3. 或在輸入框中打入**自訂指令**（例如：「加入具體範例」、「改成繁體中文口語」）
4. 點擊「**改寫**」，AI 根據指令修改並顯示結果
5. 確認後點「**套用**」取代原本內容；不滿意則點「**捨棄**」

> 💡 預設指令與自訂指令可同時使用，系統會以逗號串接後一起送給 AI。

---

### 第五步：下載與匯入知識庫

點擊右上角「**下載 Q&A**」：

| 欄位 | 說明 |
|------|------|
| **Q&A 集名稱** | 輸入本次整理的名稱，作為下載 TXT 的檔名，也作為匯入知識庫的來源標識 |
| **同時匯入至知識庫** | 開啟後可選擇現有知識庫或建立新知識庫 |

點擊「**確認下載**」後：
- 自動下載 TXT 備份到本機
- 若有開啟匯入，同步將 Q&A 推送至知識庫

---

## 重要特性

- **跨文件 Q&A 累加**：上傳新文件時可選擇「累加模式」，新產生的 Q&A 附加在既有清單後，適合分批整理大量文件
- **筆記模式預設累加**：筆記模式每次轉換皆自動累加，方便多次補充
- **瀏覽器暫存**：Q&A 資料與筆記文字自動儲存於瀏覽器，重新整理頁面或意外關閉後依然保留
- **三種模式獨立清單**：文件、筆記、SOP 三種模式各自維護獨立的 Q&A 清單，互不干擾

---

## 注意事項

- 掃描式 PDF（純圖片）無法提取文字，請使用含文字層的 PDF，或先使用 OCR Agent 處理。
- 文件包含表格時，表格會自動轉為結構化格式供 AI 理解，整理結果更準確。
- 整理結果僅供參考，建議人工審閱（並視需要使用 AI 改寫）確認後再匯入知識庫。
- 長文件（逾 20,000 字）會自動切段分批整理，各段結果依序累積顯示。
