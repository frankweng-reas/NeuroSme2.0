# KB Bot Builder 使用說明

## Overview

**KB Bot Builder** 以企業知識庫為基礎，建立 AI 問答 Bot，並部署至所需場景。

| 部署情境 | 說明 |
|---------|------|
| **對外客服 Widget** | 嵌入官網、產品頁面，讓訪客自助查詢產品資訊、退換貨政策等 |
| **內部員工 Bot** | 嵌入企業內部系統（OA、HR 入口），讓員工查詢 HR 政策、IT 規範、操作手冊 |
| **API 串接** | 透過 API Key 讓現有系統直接呼叫 Bot，不需要嵌入 Widget |

知識管理群組的三個 Agent 分工如下：
- **Doc Refiner** → 整理 PDF 為 Q&A 後匯入知識庫
- **Knowledge Base** → 管理文件、直接查詢
- **KB Bot Builder**（本 Agent）→ 以知識庫為基礎部署 Bot

---

## 功能定位

KB Bot Builder 讓管理員以公司知識庫為基礎，建立 AI Bot 並部署為嵌入式 Widget 或 API，供內部系統或外部網站使用。
建立 Bot 前，請先在 **Knowledge Base** 建立並設為「公司共用」的知識庫。

---

## 1. Bot 列表（左側欄）

左側欄列出本租戶下所有 Bot。

| 操作 | 說明 |
|------|------|
| **新增 Bot** | 點擊上方「＋」，輸入名稱後建立 |
| **切換 Bot** | 點選任一 Bot，右側立即切換至該 Bot 的設定與測試 |
| **設定** | 點「⋯ → 設定」進入 Bot 詳細設定 |
| **刪除** | 點「⋯ → 刪除」，**此操作無法復原** |

---

## 2. 對話測試（Chat）

選取 Bot 後，右側預設顯示對話測試介面。

- 直接輸入問題，測試 Bot 是否能根據知識庫正確回答。
- AI 回答附帶**來源標注**（來源：檔名），確認答案是否來自正確文件。
- 對話記錄會保留（local storage），離開頁面再回來仍可查看。
- 點擊右上角重新整理圖示可清除本段對話。

---

## 3. Bot 設定（設定）

| 設定項目 | 說明 |
|---------|------|
| **Bot 名稱** | 顯示名稱，也用於識別 Bot |
| **AI 模型** | 指定此 Bot 使用的 LLM，留空採系統預設 |
| **知識來源（KB）** | 勾選此 Bot 引用的公司知識庫（可多選） |
| **自訂系統提示詞** | 指定 Bot 的角色、語氣與回答格式 |

> 知識來源只顯示「公司共用」知識庫，個人知識庫不適用於對外 Bot。

---

## 4. 訪客對話（歷史）

此功能記錄外部用戶透過 Widget 與 Bot 的對話紀錄。

- 選取 Session 可查看完整對話內容。
- 可用於查核 Bot 回答品質、了解使用者常見問題。

---

## 5. API 整合

提供 API Key 讓外部系統直接呼叫 Bot 問答功能（不透過 Widget）。

| 操作 | 說明 |
|------|------|
| **產生 API Key** | 每個 Key 格式為 `nsk_...` |
| **撤銷 API Key** | 撤銷後該 Key 立即失效 |
| **查看使用量** | 顯示各 Key 的請求次數與 token 用量 |

**呼叫端點：**

```
POST /api/v1/public/bot/query
X-API-Key: nsk_your_key_here
Content-Type: application/json

{
  "bot_id": 1,
  "question": "請問退貨政策是什麼？"
}
```

Rate limit：每個 API Key 每小時最多 100 次。詳細規格請參閱 `/api/v1/public/docs`。

## 6. 部署（部署）

### 服務狀態

| 狀態 | 說明 |
|------|------|
| **啟用 Bot** | Bot 可正常服務；Widget 與 API 均可使用 |
| **停用 Bot** | Bot 暫停服務；Widget 與 API 均無法使用 |

### 嵌入式 Widget

1. 確認已產生 Widget Token（由 admin 操作）
2. 複製嵌入碼，貼入目標網頁 `<body>` 底部
3. Widget 即會在頁面右下角顯示聊天按鈕

### Widget 外觀

| 設定項目 | 說明 |
|---------|------|
| **標題** | Widget 頂部顯示的標題文字 |
| **主色** | Widget 介面主要顏色（十六進位色碼） |
| **介面語言** | 預設繁體中文（zh-TW） |
| **語音輸入** | 啟用後，用戶可用語音輸入問題 |
| **語音提示詞** | 語音辨識的語言提示（如：繁體中文、英文） |
| **Logo 圖片 URL** | 顯示於 Widget 頭像位置的圖片連結 |

### 撤銷 Widget Token

撤銷後，所有已嵌入的 Widget 將**立即失效**，直到重新產生並更新嵌入碼為止。
此操作僅限 admin / super_admin。

## 7. 權限說明

| 操作 | member | manager | admin / super_admin |
|------|--------|---------|---------------------|
| 查看 Bot 列表 | ✅ | ✅ | ✅ |
| 建立 / 修改 Bot | ❌ | ✅ | ✅ |
| 刪除 Bot | ❌ | ✅ | ✅ |
| 產生 / 撤銷 Widget Token | ❌ | ❌ | ✅ |
| 查看訪客對話 | ❌ | ✅ | ✅ |
| 管理 API Key | ❌ | ✅ | ✅ |

---

## 注意事項

- 刪除 Bot 將同時刪除其所有 Widget Session 與 API 使用紀錄，**無法復原**。
- 撤銷 Widget Token 不會刪除對話紀錄，但 Widget 將立即停止運作。
- Bot 啟用/停用會同時影響 Widget 與 API，請確認後再操作。
- 公司知識庫的文件如有異動（新增/刪除），Bot 下次查詢即時反映，無需重新部署。
