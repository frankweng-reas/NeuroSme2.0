# NeuroSme 產品策略：以知識管理為核心主線

> 文件建立：2026-05-10
> 用途：產品方向討論的共同參考基準

---

## 一、產品定位

**目標客群**：50–500 人的中型企業，尤其是：
- 有資安顧慮（不希望資料上雲）的製造業、醫療、法律事務所
- 有大量內部文件需要管理（HR 手冊、產品規格書、合約、SOP）
- 想部署客服或內部支援 Bot，但沒有工程師

**核心主張**：
> 讓公司的知識，變成會說話的 AI

---

## 二、產品故事線（KB 主線）

企業最常見的 AI 應用需求是：「讓 AI 能回答公司自己的問題。」
NeuroSme 提供從文件整理到 Bot 部署的完整鏈路，不需要工程師介入：

```
企業有文件（PDF、手冊、規章、Q&A）
        ↓  Doc Refiner
  自動整理為結構化 Q&A
        ↓  Knowledge Base
  建立知識庫，可即時查詢
        ↓  KB Bot Builder
  部署為 Bot（一個 URL）
        ↓
  員工 / 客戶 直接用手機問答
```

每一步都有專屬工具支撐，且彼此無縫串接，是市場上少見的端到端解決方案。

## 四、核心競爭優勢

### 1. 端到端知識管理鏈路（最強差異化）
Doc Refiner → Knowledge Base → KB Bot Builder 三件套，市面上少有平台整合得這麼完整。
企業不需要自建 RAG 基礎設施，從「整理文件」到「部署 Bot」一條龍完成。

### 2. Bot 部署零門檻
KB Bot Builder 產生一個 URL，手機即可使用，不需要工程師嵌入程式碼。
相比需要 API 串接或嵌入 JS 的競品，對 SME 門檻低很多。

### 3. 支援本地模型（資料不出境）
整合 Ollama，可在企業自有主機上運行 AI 模型，資料不傳送至外部 API。
對製造業、醫療、法律等有資安顧慮的客戶是關鍵賣點。

### 4. 不被單一 AI 廠商鎖定
支援 OpenAI、Google Gemini、Anthropic、本地 Ollama，企業可依成本與需求自由切換，不受漲價影響。

### 5. All-in-one 降低管理複雜度
員工不需分別訂閱 ChatGPT Plus、Notion AI、OCR 工具，IT 統一管理帳號、權限與費用。

## 五、現階段相對較弱的地方

| 項目 | 現況 | 影響 |
|------|------|------|
| BI Agent 使用門檻 | 需先上傳 Schema，非技術用戶不熟悉 | 影響此 agent 滲透率 |
| Email / 行事曆整合 | Writing 生成後需手動複製貼上 | 增加操作步驟 |
| KB 資料來源 | 目前僅支援 PDF / TXT | 無法直接從網址或 Google Doc 建庫 |
| Bot 使用統計 | 目前已有訪客對話歷史，但無聚合統計 | 企業無法量化 Bot 效益 |
| KB 版本管理 | 文件更新後無版本記錄 | 難以追蹤知識庫演進 |

## 六、後續強化方向（圍繞 KB 主線）

以下依建議優先順序排列：

### 短期（補強現有功能）
- [ ] **Bot 使用統計 Dashboard**：常見問題、查詢次數、無解率（讓老闆看到 ROI）
- [ ] **BI Agent 引導優化**：提供範例 Schema 與範例資料集，降低上手門檻

### 中期（擴大 KB 建庫來源）
- [ ] **URL 抓取建庫**：輸入網址，自動爬取頁面內容加入知識庫
- [ ] **Google Doc / Notion 連接器**：讓企業現有文件直接進入 KB

### 長期（生態系整合）
- [ ] **Email 整合**：Writing 生成後可直接寄出（Gmail / Outlook）
- [ ] **Webhook / Zapier 連接**：Bot 回答後可觸發外部流程（如建立工單）

## 七、銷售說法建議

**一句話定位**：
> NeuroSme 是中小企業的 AI 知識管理平台，讓公司文件變成隨時可問的 AI 助理。

**三個 why buy**：
1. **省時**：員工不用翻手冊，直接問 Bot，30 秒得到答案
2. **省錢**：一個平台取代多個 AI 訂閱，支援本地模型降低 API 費用
3. **安全**：支援本地部署，資料不出公司，符合資安要求
