## 一、產品名稱與定位

**產品名稱**：NeuroSme | Private Hub

**產品性質**：這不只是一套 AI 工具集，而是一個**企業 AI 轉型平台**，分為兩個層次：

| 層次 | 說明 |
|------|------|
| **工具層（Agent）** | 各功能 Agent，員工日常使用的 AI 工具 |
| **平台層（Platform）** | 多租戶架構、細緻權限控管、使用紀錄、模型管理，企業 IT / 管理者掌控全局 |

兩個層次缺一不可。工具層解決員工的日常需求；平台層讓管理者有能力治理、量化、並持續優化企業的 AI 使用。

**目標客群**：50–500 人的中型企業，尤其是：
- 有資安顧慮（不希望資料上雲）的製造業、醫療、法律事務所
- 有大量內部文件需要管理（HR 手冊、產品規格書、合約、SOP）
- 想部署客服或內部支援 Bot，但沒有工程師
- 正在推動 AI 轉型，需要量化成效的管理階層

**核心主張**：
> 讓公司的知識，變成會說話的 AI

## 二、產品故事線（KB 主線）

企業最常見的 AI 應用需求是：「讓 AI 能回答公司自己的問題。」
NeuroSme | Private Hub 提供從文件整理到 Bot 部署的完整鏈路，不需要工程師介入：

```
企業有文件（PDF、手冊、規章、Q&A）
        ↓  Doc Refiner
  自動整理為結構化 Q&A
        ↓  Knowledge Base
  建立知識庫，可即時查詢
        ↓  KB Bot Builder
  部署為 Bot（一個 URL）
        ↓
  員工 / 客戶 直接用手機問答
        ↓
  平台記錄所有使用行為 → AI 轉型數據
```

每一步都有專屬工具支撐，且彼此無縫串接，是市場上少見的端到端解決方案。

## 三、現有 Agent 全覽

### 知識管理群組（核心主線）

| Agent | 功能 | SME 使用情境 |
|-------|------|-------------|
| **Doc Refiner** | 將 PDF 整理為 Q&A，匯入知識庫 | 員工手冊、產品規格書、法規文件 → 快速建庫 |
| **Knowledge Base** | 管理文件、直接查詢知識庫 | 員工自助查詢 HR 規定、技術文件、公司 FAQ |
| **KB Bot Builder** | 建立 Bot，產生分享 URL，可嵌入或直連 | 官網客服 Bot、HR Support Bot、IT 支援 Bot |

### 生產力群組（附加價值）

| Agent | 功能 | SME 使用情境 |
|-------|------|-------------|
| **Chat** | 通用 AI 對話，支援附件與多模型 | 日常問答、文件摘要、替代 ChatGPT 訂閱 |
| **Writing** | 商業文書生成 + AI 局部改寫 | Email、提案、月報、公告 — 高頻文書需求 |
| **OCR / Vision** | 圖片/PDF 欄位抽取，支援批量 | 名片建檔、發票處理、表單資料擷取 |

### 其他群組

| Agent | 功能 | SME 使用情境 |
|-------|------|-------------|
| **Marketing Writer** | 行銷文案生成 | 小公司沒有專職文案，AI 輔助撰寫 |
| **Business Insight** | 自然語言查詢資料庫，生成圖表 | 老闆想看銷售數據但沒有 BI 工程師 |

### 平台管理（Admin）

| 功能 | 說明 |
|------|------|
| 多租戶架構 | 每個企業獨立隔離，資料不互通 |
| 角色權限 | member / manager / admin / super_admin 四層 |
| **Agent 存取權限控管** | 可針對每位 user 設定可使用哪些 Agent，IT 決定誰能用什麼工具 |
| **LLM 模型權限控管** | 可針對每位 user 設定可使用哪些 LLM，控管 API 費用與使用範圍 |
| **全平台使用紀錄** | 所有 Agent 的使用行為皆被紀錄，可追蹤使用量、使用者、時間 |
| LLM 模型管理 | 統一設定可用模型（OpenAI / Gemini / 本地 Ollama） |
| API Key 管理 | 每個 Bot 可獨立發行 API Key，供外部系統串接 |

## 四、核心競爭優勢

### 1. 端到端知識管理鏈路（最強差異化）
Doc Refiner → Knowledge Base → KB Bot Builder 三件套，市面上少有平台整合得這麼完整。
企業不需要自建 RAG 基礎設施，從「整理文件」到「部署 Bot」一條龍完成。

### 2. Bot 部署零門檻
KB Bot Builder 產生一個 URL，手機即可使用，不需要工程師嵌入程式碼。
相比需要 API 串接或嵌入 JS 的競品，對 SME 門檻低很多。

### 3. 支援本地模型（資料不出境）
整合 Ollama，可在企業自有主機上運行 AI 模型，資料不傳送至外部 API。
對製造業、醫療、法律等有資安顧慮的客戶是關鍵賣點。

### 4. 不被單一 AI 廠商鎖定
支援 OpenAI、Google Gemini、Anthropic、本地 Ollama，企業可依成本與需求自由切換，不受漲價影響。

### 5. All-in-one 降低管理複雜度
員工不需分別訂閱 ChatGPT Plus、Notion AI、OCR 工具，IT 統一管理帳號、權限與費用。

### 6. 細緻的 Agent × LLM 權限控管（平台優勢）
可針對個別 user 設定能使用哪些 Agent 與哪些 LLM 模型，企業 IT 可精準管控：
- 哪些員工能用高成本的 GPT-4o
- 哪些部門只開放特定 Agent
- 避免員工誤用或濫用 AI 資源

這是一般 SaaS AI 工具（個人訂閱制）做不到的企業治理能力。

### 7. 全平台使用紀錄 → AI 轉型量化數據（平台優勢）
所有 Agent 的使用行為皆被記錄，管理者可取得：
- 哪些 Agent 被最常使用、哪些沒人用
- 哪些部門 / 人員 AI 使用率高
- Token 用量趨勢與費用預估
- AI 導入前後的工作效率變化基準

這讓 NeuroSme | Private Hub 不只是工具，而是企業 **AI 轉型的量化儀表板**，
協助管理者向上匯報 AI 投資回報（ROI），推動內部持續採用。

## 五、銷售說法建議

**一句話定位**：
> NeuroSme | Private Hub 是中小企業的 AI 轉型平台，讓公司知識變成隨時可問的 AI 助理，同時讓管理者掌握每一筆 AI 使用數據。

**三個 why buy**：
1. **省時**：員工不用翻手冊，直接問 Bot，30 秒得到答案
2. **省錢**：一個平台取代多個 AI 訂閱，支援本地模型降低 API 費用，細緻權限避免資源浪費
3. **安全 + 可控**：支援本地部署，資料不出公司；Agent 與 LLM 權限細緻控管，IT 完全掌握誰能用什麼

**給管理階層的額外說法**：
> 導入 AI 不只要讓員工用，還要能**量化成效**。Private Hub 記錄所有 AI 使用行為，讓 AI 轉型投資有數字說話。
